§2 確認済み事項（現状調査結果）

2-1. 各事業所から医師指示書への直接アクセス

結論: アクセス制御は実装済み（ただし共有モデルは v5.0 で拡張）
  • 既存 consent_forms は shared_to_user_id による明示共有のみ。Worker (L2364) で cf.shared_to_user_id !== currentUser.id なら拒否
  • 退院通知は notice_recipients で受信者を明示管理
  • 第三者が URL を推測しても、本人 / 所属組織 / 共有先以外はアクセス不可
  • ただし 「複数事業所への 1:N 共有」「役割ベースの閲覧制限」は未実装 → Phase 2 で medical_document_shares として整備

2-2. アセス・指示書等の過去引継ぎ

結論: 現状未実装、v5.0 で追加が必要
  • D.assessments.push(...) はあるが、過去レコードから値をコピーする UI / API は未実装
  • 記録系（退院時共同指導・サービス担当者会議・モニタリング）も同様
  • 追加方針: 各記録フォームの上部に「前回値を引き継ぐ」ボタンを追加。同一患者の最新レコードから ADL / 服薬 / 家族構成 / 病名等の汎用フィールドを初期値として埋める（§16 参照）

§7 歯科対応

7-1. 課題

大下指示: 「歯科も診療情報の発行やカンファレンス参加など、医科のように作業、操作できるようにして。記録関係はプルダウンで選べるようにしたらいいのかな？そもそもフォーマットの内容が違うのかな？」

7-2. 結論: フォーマットは似ているが項目が異なる

項目医科歯科
診療情報提供書様式厚労省様式（一般的）歯科診療情報提供書様式（医療法施行規則・歯科特有）
必須項目主訴・既往歴・現病歴・現在の治療・処方等主訴・口腔内所見・補綴物状態・歯周ポケット・X線所見等
カンファレンス退院時共同指導・サービス担当者会議同左（歯科医師の参加機関種別が必要）→ §9 で対応済み（プルダウン「歯科診療所」追加）
指示書訪問看護指示書等歯科訪問診療指示書（歯科衛生士向け）

7-3. 実装方針

  • doc_kind に dental_referral, dental_instruction を追加（既に §3-1 反映）
  • 診療情報提供書フォーム作成時、「医科 / 歯科」をユーザー選択し、項目フィールドを切り替え（テンプレで対応・別 doc_kind）
  • 退院時共同指導の機関プルダウンに「歯科診療所（訪問歯科含む）」を追加（5/26 完了）
  • 歯科診療所 (clinic_dent) ロールで院内アカウント発行可能に

7-4. 記録フォームのプルダウン

「記録関係はプルダウンで選べるようにしたら？」への回答: 診療科プルダウンで初期値を切り替える方式を採用。フォーム自体は別 doc_kind だが、UI 上はプルダウン操作で適切なテンプレが開く。

診療情報提供書の取扱方法資料は別途添付予定。受領後、以下を決定する:

  1. 記載必須項目（基本様式 vs 大下提案フォーマットの差分）
  2. 署名フローの厳密さ（システム内署名で十分 / HPKI 必須 等）
  3. 受領機関への自動共有ルール
  4. 診療情報提供料（B009: 250点等）の算定根拠化

5/28 までに資料が間に合えば実装、間に合わなければ 6 月以降に持ち越し。

§12 録音・録画ダウンロード

  • app.html 側: 通話終了後の画面に「録画 / 録音ダウンロード」ボタンを追加（Whereby API で取得 or ファイルアップロード）
  • ダウンロードしたデータを signed_documents に「録画ファイル」として登録（doc_kind 追加: call_recording）
  • 関係者への共有は §4 共有基盤を流用

§13 広告システム（段階提供）

13-1. 完了内容（5/26）

5/26 完了: LP（index.html）の料金セクション直後に「広告との付き合い方を、選べます。」セクション追加。3 プラン:
  1. PLAN 01 広告非表示プラン（利用者向け有料オプション・料金未定）
  2. PLAN 02 広告掲載プラン（広告主向け・料金未定）

13-2. 料金未定の理由

大下指示「広告無し（月額いくら？）。広告掲載料（月額いくら？）」に対し、相場調査が必要。参考:

サービス広告非表示プラン広告掲載料
YouTube Premium¥1,280/月CPM ¥300-1000
Spotify Premium¥980/月CPC ¥30-100
類似BtoBアプリ¥500-2000/月¥30,000-100,000/月
医療広告ガイドラインの制約: 医療機関に対する広告は厚労省「医療広告ガイドライン」に基づく審査が必要。誇大広告・効果保証・体験談等は禁止。広告審査機能の実装が必要（半年後タスク）。

13-3. 地域別ターゲティング

登録時に取得した「所属事業所所在地」を元に、都道府県・市区町村単位で出し分け。大阪の事業所が東京の利用者に表示されないよう、絶対的にフィルタ。

§16 アセス・指示書の過去引継ぎ

16-1. 大下指示

アセスや指示書等の記録計は、過去の内容を引き継げるようになっているか？

16-2. 現状調査結果

未実装。現状は新規作成時に空フォームが開く。

16-3. 実装方針（5/27-28）

  • 各記録フォーム（アセスメント・退院時共同指導・サービス担当者会議・モニタリング・指示書）の上部に「前回値を引き継ぐ」ボタンを追加
  • クリック時、同一患者の同種類最新レコードを取得し、以下のフィールドを初期値として埋める:
    • ADL（食事・排泄・入浴・移動・認知）
    • 服薬リスト
    • 家族構成・主介護者
    • 病名・既往歴
    • 医療処置（経管栄養・カテーテル等）
  • 引き継いだ項目は薄黄色背景で「前回値」マークを表示、編集すると通常背景へ
  • 引き継ぎ元レコードのIDを copied_from_record_id に保存（監査ログ化）
